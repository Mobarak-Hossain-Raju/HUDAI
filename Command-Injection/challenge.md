# Security Challenges Report

## High Severity Challenges
- Exploitation required identifying vulnerable parameters manually.
- Input filtering occasionally blocked payload execution.
- Server responses varied during repeated exploitation attempts.

## Medium Severity Challenges
- Session expiration interrupted testing continuity.
- Payload encoding adjustments were necessary for successful exploitation.
- Verification of vulnerability impact required additional validation.

## Low Severity Challenges
- Organizing screenshots and documentation required additional formatting.
- Matching evidence with exploitation stages consumed extra time.

## Security Assessment Observation
The assessment identified multiple weaknesses caused by improper validation, insecure request handling, and insufficient filtering mechanisms. The testing process remained controlled and limited to authorized educational laboratory environments.
